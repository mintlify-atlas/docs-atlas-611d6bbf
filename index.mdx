<div className="mt-16 mb-16 max-w-5xl mx-auto px-6">
  <h2 className="text-3xl font-bold text-gray-900 dark:text-white mb-4">Quick Start</h2>
  <p className="text-lg text-gray-600 dark:text-gray-400 mb-8">
    Get started with the Cross-Media Measurement API in just a few steps.
  </p>

  <Steps>
    <Step title="Understand the Architecture">
      The Cross-Media Measurement API is a resource-oriented gRPC API using Protocol Buffers. It enables privacy-preserving measurements through multi-party computation across data providers.
      
      Read the [Core Concepts](/concepts) to understand the resource model and architecture.
    </Step>

    <Step title="Set Up Authentication">
      Authentication uses OpenID Connect and X.509 certificates for digital signatures. You'll need to create an account and obtain credentials.

      ```bash
      # The API uses certificate-based authentication
      # See the authentication guide for details
      ```

      See the [Authentication Guide](/guides/authentication) for complete setup instructions.
    </Step>

    <Step title="Create Your First Measurement">
      Once authenticated, you can create measurements that compute reach, frequency, or impressions across multiple data providers while preserving privacy.

      ```protobuf
      message Measurement {
        string name = 1;
        string measurement_consumer_certificate = 2;
        MeasurementSpec measurement_spec = 3;
        State state = 4;
        Result result = 5;
      }
      ```

      Follow the [Creating Measurements Guide](/guides/creating-measurements) to get started.
    </Step>

    <Step title="Explore the API">
      The API provides comprehensive services for measurements, event groups, requisitions, model management, and panel matching.

      Browse the [API Reference](/api/overview) to explore all available services and methods.
    </Step>
  </Steps>
</div>
