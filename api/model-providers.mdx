---
title: Model Providers API
description: API reference for managing Model Provider resources
---

The Model Providers API allows you to retrieve and list model providers in the Cross-Media Measurement system. Model providers are parties that build VID models.

## Resource

A `ModelProvider` represents the party that builds VID Models.

### ModelProvider

<ResponseField name="name" type="string" required>
  Resource name with pattern `modelProviders/{model_provider}`
</ResponseField>

<ResponseField name="display_name" type="string">
  Display name of the ModelProvider
</ResponseField>

## Methods

### GetModelProvider

Returns the `ModelProvider` with the specified resource key.

<ParamField path="name" type="string" required>
  Resource name of the ModelProvider to retrieve.
  
  Format: `modelProviders/{model_provider}`
</ParamField>

**Response**

Returns a `ModelProvider` resource.

**Example Request**

```proto
GetModelProviderRequest {
  name: "modelProviders/123"
}
```

### ListModelProviders

Lists `ModelProvider` resources.

<ParamField query="page_size" type="int32">
  The maximum number of ModelProviders to return. The service may return fewer than this value.
  
  - If unspecified, at most 50 ModelProviders will be returned
  - Maximum value is 1000; values above 1000 will be coerced to 1000
</ParamField>

<ParamField query="page_token" type="string">
  A page token received from a previous `ListModelProviders` call. Provide this to retrieve the subsequent page.
  
  When paginating, all other parameters must match the call that provided the page token.
</ParamField>

**Response**

<ResponseField name="model_providers" type="ModelProvider[]">
  List of ModelProvider resources
</ResponseField>

<ResponseField name="next_page_token" type="string">
  A token that can be sent as `page_token` to retrieve the next page. If omitted, there are no subsequent pages.
</ResponseField>

**Example Request**

```proto
ListModelProvidersRequest {
  page_size: 50
}
```
